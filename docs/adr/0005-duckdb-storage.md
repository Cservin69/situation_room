# ADR 0005 — DuckDB as the storage engine

**Status**: Accepted
**Date**: 2026-04-19

## Decision

DuckDB from day one. Not SQLite. Not Postgres.

## Rationale

[To be written together. Columnar storage for analytical queries, ships as
single file (instant clone), spatial extension for geometry queries, native
Parquet interop, embedded model = zero infra for OSS users.]
